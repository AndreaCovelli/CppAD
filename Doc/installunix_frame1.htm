Unix Installation and Testing

Download Current Version
Download the current version of the file CppAD.unix.tar.gz

Extraction
Use the command
     tar -xvzf CppAD.unix.tar.gz
 to decompress and extract the unix format version into the distribution directory
     cppad-yy-mm-dd
The value yy-mm-dd is the corresponding version of CppAD.

User Documentation
The Doc sub-directory of the distribution directory contains user documentation for CppAD. The files Doc/*.htm are in HTML format. If your browser supports XHTML+MATHML format, an easier to read copy is available in the files Doc/*.xml.

Developer Documentation
The Developer sub-directory of the distribution directory contains developer documentation for CppAD.

Configure
Enter the directory created by the extraction and execute the command:
     ./configure \
          [ prefix=prefix ] \
          [ ADOLC_DIR=AdolcDir ] \
          [ FADBAD_DIR=FadbadDir ] \
          [ --with-stdvector ]  [ BOOST_DIR=BoostDir ] \
          [ CPP_ERROR_WARN=CppErrorsAndWarnings ] \
          [ --with-profiling ]
where the entries between the brackets [ and ] are optional, the brackets are not included, and the entires in italic are replaced values that you choose.

Prefix
The default value for prefix is $HOME; i.e., by default the CppAD include files will install in
     $HOME/include/CppAD
and the CppAD library is
     $HOME/lib/libCppAD.a
If you want to install elsewhere, you will have to use this option. For example, if you have permission to write into the directories /usr/local/include and /usr/local/lib,
     ./configure --prefix=/usr/local
 will set up for installing the CppAD include files in the directory
     /usr/local/include/CppAD
and the CppAD library will be
     /usr/local/lib/libCppAD.a
This is the standard location for such files on many systems. If prefix is not a standard location for your system, you will need the compiler switches
     -I prefix/include -L prefix/lib
on the command line when you compile and link your programs with CppAD.

Vector Class
The default definition for CppADvector is CppAD::vector .

If the command line argument --with-stdvector is specified, this definition is replaced by std::vector. In this case BoostDir must not be specified.

If the command line argument
     BOOST_DIR=BoostDir
is present, --with-stdvector must not also be present. The value of BoostDir must be such that
     BoostDir/boost/numeric/ublas/vector.hpp
is a valid reference to the file vector.hpp. This replaces the default definition of CppADvector with
     boost::numeric::ublas::vector
 See boost .

C++ Errors And Warnings
If the command line argument CppErrorsAndWarnings is present, it specifies compiler error and warning flags. For example,
     CPP_ERROR_AND_WARN="-Wall -ansi"
would specify that error and warning flags -Wall and -ansi should be included in all the C++ compile commands. The error and warning flags chosen must be valid options for the C++ compiler. The default value for CppErrorsAndWarnings is the empty string.

Profiling
See the Profiling sub-heading under the Speed tests.

Building All Tests and Examples
The following command will build all of the examples and tests:
     make
Getting Started
The following commands will run the getting started example:
     cd GetStarted
     ./GetStarted
Extensive Examples and Testing
The following commands will an extensive set of examples and tests:
     cd Example
     ./Example
 The following commands will execute an example and test of printing values during forward mode calculations:
     cd PrintFor
     ./PrintFor
More Correctness Testing
The following commands run some more correctness tests:
     cd TestMore
     ./TestMore
Speed Testing
The following commands run a set of speed and tests:
     cd Speed
     ./Speed
Profiling
If the command line argument --with-profiling is specified, a program called Speed/Speed_p is generated. In this case, the following commands will profile the speed tests:
     cd Speed
     ./Speed_p
     gprof -b Speed_p
 If you are using a windows operating system, you may have to replace Speed_p by Speed_p.exe in the command above. Profiling is only support for the GNU C++ compiler.

Compare CppAD with Adolc
If you have ADOLC installed on your system, you can specify a value for AdolcDir in the configure command line. The value of AdolcDir must be such that
     AdolcDir/include/adolc/adouble.h
is a valid way to reference adouble.h. In this case you can compare the correctness and speed of your version of CppAD with version 1.9 of Adolc. The commands
     cd Adolc
     ./Example
     ./Speed
will run both the speed and correctness tests.

If you are using Cygwin, you will have to add to following lines to the file .profile in your home directory:
     PATH=AdolcDir/bin:$PATH
     export PATH
in order for Adolc to run properly. If AdolcDir begins with a disk specification, you must use the Cygwin format for the disk specification. For example, if d:/ADOLC19 is the proper directory, /cygdrive/d/ADOLC19 should be used for AdolcDir.

Compare CppAD with Fadbad
If you have FADBAD++ installed on your system, you can specify a value for FadbadDir. It must be such that
     FadbadDir/FADBAD++/badiff.h
is a valid reference to badiff.h. In this case you can compare the correctness and speed of your version of CppAD with FADBAD. The commands
     cd Fadbad
     ./Example
     ./Speed
will run both the speed and correctness tests.

Final Installation
Once you are satisfied that the tests are giving correct results, you can install CppAD into easy to use directories by executing the command
     make install
 This will install CppAD in the location specified by prefix .
