This:  section, frame
Description
Suppose that left is an AD<Base> object, and right is an int, Base or AD<Base> object. In this case
     left /= right
returns an AD<Base> object where /= has the same interpretation as for the Base type. In addition, this must be the same as
     left = left / right

Operands
Note the value of the left operand is changed and the value of the right remains unchanged. If the left operand has value zero and is a parameter or a Base operand, the result is a parameter. Otherwise, if the left or right operand is a variable, the result is a variable .

If the right operand is exactly equal to one and it is a parameter or a Base operand, the result has the same derivative information as the left operand and no extra space on the tape is required for this operation.

Assumptions
If the /= operator is used with an AD<Base> object, it must be a computed assignment operator for the base type Base. In addition, if  f and  g are Base functions with the same range dimension,  \[
     \D{[ f(x) / g(x) ]}{x} = 
          [1/g(x)] * \D{f(x)}{x} - [f(x)/g(x)^2] * \D{g(x)}{x} 
\] 

Example
The file DivEq.cpp contains an example and a test of these operations. It returns true if it succeeds and false otherwise.
