Description
The SpeedTest function executes a speed test for various sized problems and reports the results on standard output; i.e. std::cout. The size of each test problem is included in its report (unless first is equal to last).

Repeat Factor
It is important to separate small calculation units and test them individually. This way individual changes can be tested in the context of the routine that they are in. On many machines, accurate timing of a very short execution sequences is not possible. In addition, there may be set up time for a test that we do not really want included in the timing. For this reason SpeedTest automatically determines how many times to repeat the section of the test that we wish to time.

Include
The file SpeedTest.h contains the prototype for the SpeedTest function. This file is included by CppAD/CppAD.h but it can also be included separately with out the rest of the CppAD routines.

Linkage
The CppAD Unix installation compiles this function and places the object in the CppAD.a library; i.e., you will need the command line argument -lCppAD when you link to this routine under Unix. If you are using Microsoft visual studio, you will have to include the file lib/SpeedTest.cpp in your set of project source files.

Test
The argument Test is a function such that
     Test(size, repeat)
repeats a test for a specified size and number of times and then returns a name for the test as a string. The arguments size and repeat (to Test) have type size_t. Only one name is reported per call to SpeedTest and it is the return value from the a call to Test where size is equal to first and the value of repeat is not specified. If the name returned by Test is the empty string, the name of the test is not reported by SpeedTest.

first
The argument first has type size_t and specifies the size of the first test problem reported by this call to SpeedTest.

last
The argument last has type size_t and specifies the size of the last test problem reported by this call to SpeedTest.

inc
The argument inc has type int and specifies the increment between problem sizes; i.e., all values of size in calls to Test are given by
     size = first + j * inc
where j is a positive integer. The increment can be positive or negative but it cannot be zero. The values first, last and inc must satisfy the relation  \[
     inc * ( last - first ) \geq 0
\] 

rate
The value displayed in the rate column on std::cout is defined as the value of repeat divided by the corresponding elapsed execution time in seconds. The elapsed execution time is measured by the difference in
     (double) clock() / (double) CLOCKS_PER_SEC
 in the context of the standard <ctime> definitions.

Errors
If one of the restrictions above is violated, the error reporting macro CppADUsageError is used to report the error. You can redefine this action using the instructions in CppADError
